# 产品概述

## 1. 项目愿景 (Vision)

_用一句话概括：我们要为谁，提供什么核心价值？_

> 为政策研究员和经济学开发者提供一个基于 Vue 3 的 RAG 演示应用，通过 MCP 连接本地文件夹，实现对经济研究文档的语义检索和问答。

## 2. 核心价值 (Value Proposition)

- **用户价值**：让政策研究员和经济学开发者能够快速、准确地从大量经济研究文档中获取所需信息，提高研究效率和决策质量。
- **商业/技术价值**：展示 RAG 技术在经济研究领域的应用潜力，为相关企业和机构提供技术参考和解决方案。

## 3. 核心板块 (Core Modules)

_将系统拆解为关键的顶层模块（建议 2-4 个核心板块）。_

### 3.1 文档管理模块

- **描述**：负责连接本地文件夹，管理经济研究文档（PDF/Markdown）的导入、索引和存储。
- **关键特性**：
  - 通过 MCP 协议连接本地文件夹
  - 支持 PDF 和 Markdown 文档格式
  - 自动提取文档内容并建立索引

### 3.2 语义检索模块

- **描述**：基于文档内容进行语义理解和检索，返回与用户查询相关的文档片段。
- **关键特性**：
  - 支持自然语言查询
  - 基于语义相似度进行检索
  - 提供相关度排序和结果高亮

### 3.3 问答交互模块

- **描述**：基于检索到的文档内容，生成准确、相关的回答，与用户进行自然语言交互。
- **关键特性**：
  - 基于 RAG 技术生成回答
  - 提供回答来源引用
  - 支持多轮对话

## 4. 目标用户 & 核心场景 (User & Scenes)

### 4.1 目标用户

- **政策研究员**：需要从大量经济研究文档中提取信息，分析政策影响和经济趋势。
- **经济学开发者**：需要了解 RAG 技术在经济研究领域的应用，开发相关工具和系统。

### 4.2 核心场景 (Key Use Cases)

- **场景 1**：政策研究员在准备政策建议时，通过自然语言查询，从大量经济研究文档中快速获取相关信息，提高研究效率。
- **场景 2**：经济学开发者在开发经济分析工具时，通过本应用了解 RAG 技术的实现方式和应用效果，为自己的项目提供参考。

## 5. 关键业务流程 (Core Workflows)

_描述系统最核心的 1-2 条业务链路，这决定了技术架构的复杂度。_

> 用户输入查询 -> 系统通过 MCP 连接本地文件夹 -> 检索相关文档 -> 提取文档内容 -> 生成回答 -> 展示结果和来源

## 6. 验收标准 (Acceptance Criteria)

_项目第一阶段结束的具体标志是什么？（用于防止需求蔓延）_

- **功能完备性**：核心流程跑通，能够连接本地文件夹，实现对 PDF 和 Markdown 文档的语义检索和问答。
- **技术指标**：页面加载时间 < 2s，检索响应时间 < 5s，回答准确率 > 80%。

## 7. 约束条件 (Constraints)

_非功能性的硬性限制。_

- 基于 Vue 3 框架开发
- 通过 MCP 协议连接本地文件夹
- 支持 PDF 和 Markdown 文档格式
- 开发周期 < 4周

## 8. 项目边界 (Out of Scope)

_明确本阶段**不做**什么，避免需求蔓延。_

- 暂不支持多语言
- 暂不开发移动端 App
- 暂不支持文档编辑功能
- 暂不实现用户认证和权限管理
